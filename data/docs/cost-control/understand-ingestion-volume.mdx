This guide walks you through identifying which signal, service, and environment is responsible for your ingestion volume, so you know where to focus before making any changes.

## Step 1: Import the Cost Meter Dashboard

<Admonition type="note">
The **[Cost Meter module](https://signoz.io/docs/cost-meter/overview/)** (accessible from the sidebar under **Cost Meter**) is built into SigNoz, no import required. The **[Cost Meter Dashboard Template](https://signoz.io/docs/dashboards/dashboard-templates/cost-meter/)** below is a separate importable JSON dashboard that provides a service- and environment-level breakdown not available in the built-in module. Use the built-in module for a quick overview; import this template when you need to drill down by service or environment.
</Admonition>

Navigate to **Dashboards > New Dashboard > Import JSON** and import our official template:

**[Cost Meter Dashboard Template](https://signoz.io/docs/dashboards/dashboard-templates/cost-meter/)**

1. Set your timeframe (e.g., `Last 1 week`) to match the time period you are investigating.
2. Review the breakdown of volumes by signal.
3. Review the breakdown by `service.name` and `deployment.environment`.
4. **Conclusion:** _e.g., "Traces generated by the `payment-worker` service in the `Production` environment is the highest driver."_

<Admonition type="caution">
**Metrics: Use Sample Count, Not Size:** When reviewing metrics cost in the Cost Meter Dashboard, focus on the sample count panel (`signoz.meter.metric.datapoint.count`). Metric ingestion in SigNoz is measured by sample count, not bytes. Size-based panels in the dashboard apply to logs and traces only.
</Admonition>

## Step 2: Drill Down with the Ingestion Analysis Dashboard

For a granular breakdown of ingestion by individual metric names, span operation names, and log severities, import the **[Ingestion Analysis Dashboard](https://signoz.io/docs/dashboards/dashboard-templates/signoz-ingestion-analysis/)**.

This dashboard provides more granularity than the Cost Meter Dashboard. For example, it identifies which specific span operation or metric name within a service is responsible for the volume.

<Admonition type="warning">
**15-Minute Window:** The Ingestion Analysis Dashboard shows only the last 15 minutes of data. If you are investigating a historical time window, this dashboard does not cover it. Use the Explorers below.
</Admonition>

## Steps 3–5: Historical Analysis via Explorers

If the Ingestion Analysis Dashboard does not cover the time window you need, use native Explorers.

### Step 3: Historical Logs

Navigate to **Logs** and swap to the **Table** tab. If the Cost Meter Dashboard identified a specific service, filter by `service.name` and `deployment.environment`. Group by `severity_text` over your target timeframe to see if `DEBUG` or `INFO` lines are dominating the output.

<Figure src="/img/docs/cost-control/historical-logs-severity-breakdown.webp" alt="Logs Explorer grouped by severity_text showing log count per severity level" caption="Logs Explorer: group by severity_text to identify which log levels dominate volume" />

### Step 4: Historical Traces

Navigate to **Traces > Trace Explorer** and swap to the **Table** tab. Filter by your target environment/service, group by `name` (the span's operation name), and sort by total count. This reveals which span operations are generating the highest volume.

<Figure src="/img/docs/cost-control/historical-trace-operations-breakdown.webp" alt="Trace Explorer table view grouped by span operation name showing count per operation" caption="Trace Explorer: group by name and sort by count to find the highest-volume span operations" />

### Step 5: Historical Metrics

Navigate to **Metrics > Metrics Explorer**. The list view shows all ingested metrics with their sample count and time series count.

1. Set the aggregate to **Samples** and select your target time window.
2. Sort by sample count. This shows which metric names are consuming the most samples and by how much.

<Figure src="/img/docs/cost-control/historical-metrics-samples.webp" alt="Metrics Explorer list view sorted by sample count" caption="Metrics Explorer: sort by sample count to identify the highest-volume metrics" />
