Your on-call shift is ending. The next engineer is about to take over, and they need to know what happened during your shift.

Instead of manually writing up a handoff doc, you ask your AI assistant, connected to the SigNoz MCP, to generate a summary from recent alert history.

<YouTube id="7jROwJ0Ved0" mute={false} />

## Prerequisites

- Connect your AI assistant to SigNoz using the [MCP Server guide](https://signoz.io/docs/ai/signoz-mcp-server/).
- Make sure you have alerts configured in SigNoz. See [Alerts](https://signoz.io/docs/alerts/) if you haven't set this up.

## Generate the Handoff Summary

Ask your AI assistant to pull the alert history and format it as a handoff brief:

```text
Get the alert history for the last 48 hours. For each alert that fired, tell me which service was affected, when it fired, peak severity, and whether it's resolved or still open. Format as a handoff summary.
```

Your assistant will use the SigNoz MCP server to fetch the alert history and generate a structured handoff. Here's an example response:

```text
All 3 alerts currently firing (severity: error):

**Frontend Service**: 65 flaps in 48h. Peak error rate 100%. OPEN.

**Payment Service**: 119 flaps in 48h (one every 24 min). Peak error rate 50%. OPEN.

**Checkout Service**: 44 flaps in 48h. Peak error rate 100%. OPEN.

All three alerts have been flapping continuously—error rates dip below threshold
briefly then spike back up. None have cleanly resolved. The high flap counts suggest
the underlying issue isn't being fixed, just intermittently masked by low traffic.
```

## Under the Hood
</ToggleHeading>

During this investigation, the MCP server called these tools:

| Step | MCP Tool | What It Did |
|------|----------|-------------|
| 1 | `signoz_list_alerts` | Retrieved currently firing Alertmanager alert instances |
| 1 | `signoz_get_alert_history` | Fetched the complete alert state transition history for the 48-hour window to calculate flap counts and identify patterns |
| 1 | `signoz_get_alert` | Retrieved detailed information for each alert including severity, conditions, and current status |

</details>
